    Introduction PF1

    As we embark on our journey as programmers, we will use this lesson to lay the foundations of some core and recurrent concepts and structures within Python. The concepts covered in these lesson materials are among the most universal, transferrable and scalable, and will underscore everything from printing your name, through to machine learning, in later modules.

    At the most fundamental level, we will explore input and output operations in Python by means of the input() and print() functions, progressing through to understanding different data types, and how we can store them in recallable structures termed variables. We will also learn how to annotate code in comments, which the Python interpreter knows to ignore, for explaining constructs and lines of code to other users (and our future selves).

    As you progress through the lesson, you will encounter logical operations and conditional statements - powerful ways of exploring how blocks of code can be executed in the presence or absence of certain conditions specified by the user. And lastly, we will look at errors - what the common ones mean, and how to write and define your own, for when your code encounters a stumbling block that it didn't quite expect.

    These lessons may seem long, and we don't expect that you memorise every line of them. Rather, we hope that you will be able to understand the concepts and principles, guiding you through the code that we will explore as the course progresses.

    Learning overview

    This table contextualises, and reinforces connections between concepts taught in this lesson. It can be a useful aid to glance over once you have read through the lesson; helping you to connect concepts, techniques and definitions together, meaningfully.

    Concept Synergy Syntax & Functions Objective / Application Common Pitfalls / Notes
    Input / Output Links directly with variables (storing user input) and conditionals (responding to input) print(), input() Display information to the user or collect typed input Forgetting that input() returns a string; requiring type conversion
    Variables & Data Types Needed before performing logic or maths; interacts with input and output x = 5, name = "Oliver" Store and reuse data across code Mixing types (e.g. adding string to int)
    Typecasting Converts data between input/output and calculations int(), float(), str(), bool() Transform data types for calculations or display Losing precision during conversion
    Comments Improve code readability across all modules # This is a comment Annotate logic to explain function or intent Forgetting to comment complex code
    Strings & Formatting Often printed as program output; key to I/O clarity Escape sequences, f"{var}" Combine variables with readable output Misusing quotes or forgetting escape characters
    Mathematical Operators Foundation for numeric computation and logical testing +, -, , /, //, %, *, abs() Perform arithmetic operations Confusing / and // (true vs floor division)
    Boolean Logic Connects mathematical expressions to decision-making True, False, and, or, not Evaluate truth values for control flow Assuming non-Boolean values can’t act as True/False
    Comparison Operators Feed into conditional statements ==, !=, <, >, <=, >= Compare values for equality or ordering Mixing = and ==
    Conditional Statements Depend on Boolean and comparison logic if, elif, else Execute code conditionally Misaligned indentation or missing colons
    Nested Conditions Combine multiple logical layers Indented if blocks Build multi-level decision structures Logical redundancy or contradictory checks
    Error Handling Essential when using input, conditionals, or file operations try, except, NameError, TypeError, etc. Gracefully manage runtime errors Catching overly broad exceptions
    Exception Hierarchy Connects to robustness of code try: ... except ExceptionType: Target specific issues for debugging Missing specific exception names
    Debugging / Reading Errors Reinforces every prior section Reading tracebacks Understand where and why code failed Ignoring the first line of an error message
