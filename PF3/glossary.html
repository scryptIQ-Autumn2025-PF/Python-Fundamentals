Python Fundamentals 3 - Glossary

Glossary PF3
.py extension
The file extension for Python files; any file with this extension is considered a module that can be imported.
Alias
An alternative name given to a module or package when importing using the 'as' keyword (e.g., import pandas as pd).
Associative arrays/memories
The generic name in other coding languages for data structures like dictionaries that map keys to values.
Base Python
The core Python language including fundamental syntax, data types (strings, lists, dictionaries), control structures (if/else, loops), and basic built-in functions that are immediately available without importing anything.
break
A statement that immediately terminates a loop, even if there are items left to process or the loop condition is still True.
Built-in modules
Additional modules that come pre-installed with Python as part of the standard library, but must be imported to use (e.g., math, datetime, os).
Child dictionary
A dictionary nested within another (parent) dictionary as a value.
continue
A statement that skips the remaining code in the current loop iteration and moves immediately to the next iteration.
Current directory
The folder where your Python script or notebook is located; Python searches here first when looking for modules to import.
Dictionary
A data structure that maps keys to values, allowing you to store and retrieve data using descriptive keys rather than numerical indices.
Dot notation
The syntax for accessing functions or attributes from a module using a dot (e.g., math.sqrt() uses the sqrt function from the math module).
enumerate()
A built-in function that provides both the index position and the item itself during iteration over a sequence.
Environment
An isolated installation of Python with its own set of installed packages, preventing conflicts between different projects' dependencies.
for loop
A loop structure that iterates over each item in a sequence (such as a string, list, dictionary, or range), executing a block of code once per item.
Hierarchical organisation
A structured arrangement of data with multiple levels of depth, such as nested dictionaries within dictionaries.
Immutable
A property of data types that cannot be changed after creation; dictionary keys must be immutable types like strings, numbers, or tuples.
Import
A Python statement that loads modules or specific functions from modules into your current programme, making their functionality available for use.
Import statement
The Python statement used to load modules or packages (e.g., import math or from math import sqrt).
Indentation
Python's use of whitespace (tabs or spaces) to define code blocks and scope; essential for structures like loops, conditionals, and functions.
Indexing
The process of accessing values in a dictionary using keys within square brackets.
Insertion order
The order in which items were added to a dictionary; maintained and displayed in Python 3.7+.
Iterable
An object that can be looped over, such as strings, lists, dictionaries, tuples, sets, or ranges.
Iteration
The process of repeating a set of instructions multiple times, typically by looping through items in a sequence.
Item
A key-value pair within a dictionary (e.g., "temperature_celsius": 30).
Key
A unique identifier used to access a value in a dictionary (e.g., "temperature_celsius").
KeyError
An error raised when trying to access a dictionary key that doesn't exist.
Library
A general term used in programming to refer to collections of code; in Python this typically refers to modules or packages.
List comprehension
A concise, single-line syntax for creating new lists by combining an expression, a for loop, and an optional condition within square brackets.
Local modules
Python files created by you and stored in your current working directory that can be imported into other programmes.
Loop
A programming structure that enables automatic repetition of code, such as for loops and while loops.
Loop variable
The variable that holds the current item during each iteration of a loop (e.g., 'gene' in 'for gene in genes').
Module
A single Python file containing code (functions, classes, variables) that can be imported and used in other Python programmes.
Mutable
A property of data types that can be modified after creation; dictionaries are mutable, allowing items to be added, modified, or removed.
Nested dictionary
A dictionary that contains other dictionaries as values, creating hierarchical data structures.
Package
A collection of related modules organised in a directory structure, allowing for hierarchical organisation of code and functionality.
Package manager
A tool for installing and managing Python packages; examples include pip, conda, and uv.
Parent dictionary
The outer dictionary that contains nested (child) dictionaries as values.
pip
The standard Python package manager used to install third-party packages from PyPI via the command line.
PyPI
Python Package Index; the official repository where Python packages are published and can be downloaded from using pip.
range()
A built-in function that generates a sequence of numbers for iteration, accepting start, stop, and step arguments (stop is exclusive and step defaults to 1).
Site-packages directory
The location where third-party packages are installed; Python searches here after checking the current directory and standard library.
Standard library
The collection of built-in modules that come pre-installed with Python, providing extended functionality beyond base Python (e.g., math, datetime, json).
Sub-module
A module within a package; accessed using dot notation (e.g., email.mime is a sub-module of the email package).
Third-party modules
Packages created by external developers that must be installed separately (typically using pip) before they can be imported (e.g., pandas, numpy, matplotlib).
Value
The data associated with a key in a dictionary (e.g., 30).
while loop
A loop structure that continues to execute a block of code as long as a specified condition remains True.
