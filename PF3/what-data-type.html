    Choosing the Right Data Type PF3

    Learning Objectives

    1 When to choose a data `type`

    Introduction

    Now that we have covered all the common data types in base Python, it is worth having a quick overview.

    Choosing the correct data type for your project is crucial, and can require careful thought, as it can have downstream effects on your code and output. But don't fret too much, as it can always be changed and people are constantly arguing a singular case, which is not always the only way. It's always worth doing a quick search on the internet (or now, using a GenAI of your choosing) to see what others in your field are doing.

    Data Type Overview

    Data type Examples When to use it When not to use it
    Boolean (bool) True, False If there are only two possible states, true or false If your data is not binary
    Integer (int) 1, 0, -1023, 42 Countable, singular items. How many patients are there, how many events did you record, how many variants are there in the sequence If doubling or halving the value would not make sense: do not use for e.g. patient IDs, or phone numbers. If these are integers you might accidentally do maths on the value.
    Float (float) 123.49, 3.14159, -3.33334 If you need more precision or partial values. Recording distance between places, height, mass, etc. When whole numbers are more appropriate or when precision isn't needed
    Strings (str) 'patient_12312', 'Jane Doe', '火锅' To store free text, identifiers, sequence IDs, etc. If it's truly a numeric value you can do calculations with..
    List / Array (list) ['A', 1, 3.4, ['Nested']] If you need to store a list of items, like sequences from a file. Especially if you're reading in a table of data from a file. If you want to retrieve individual values, and there are clear identifiers it might be better as a dict.
    Tuple (tuple) ("chr1", 12345, 67890), (30, 37, 42) For fixed collections of related data that shouldn't change, like genomic coordinates (chromosome, start, end) or experimental conditions. When you need ordered, immutable data. When you need to modify the collection after creation, or when the number of items varies.
    Set (set) {"ACTG", "GCTA", "TTAG"}, {1, 2, 3, 5, 8} For collections of unique items where you need to check membership quickly, remove duplicates, or perform set operations (union, intersection). E.g. unique gene names, patient IDs. When order matters, when you need duplicates, or when you need to access items by index.
    Dictionary / Associative Array / map (dict) {"yeast_species": "Saccharomyces cerevisiae", "colony_count": 127, "temperature_celsius": 30, "growth_medium": "YPD"} When you have identifiers for your data, and want to look them up by that value. E.g. looking up sequences by an identifier, or data about students based on their name. Counting values. If you just have a list of items without identifiers, it makes more sense to just use a list.
