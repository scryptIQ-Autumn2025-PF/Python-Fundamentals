Python Fundamentals 4 - Glossary

Glossary PF4
Append mode ("a"): A file mode that adds new content to the end of an existing file without overwriting it.
Binary mode: A file mode (`"rb"`, `"wb"`) used when reading or writing non-text files such as images, audio, or encoded data.
Call (function call): The action of executing a function using its name followed by parentheses, optionally containing arguments.
Default argument: A parameter that automatically takes a predefined value when no value is provided by the caller.
Docstring: A triple-quoted string (`""" """`) placed at the start of a function, module, or class describing its purpose, inputs and outputs.
Encoding: A system that converts text to bytes and back again. Common encodings include UTF-8, used when reading or writing text files.
Exception: An error detected during execution (e.g., `FileNotFoundError`, `IsADirectoryError`) that can be handled using `try`/`except`.
File handle: The object returned by `open()`, representing an open file. It can be used to read from or write to that file.
File mode: A string such as `"r"`, `"w"`, `"a"`, `"rb"` or `"wb"` that tells Python how to open a file (read, write, append, binary).
File path: The location of a file on your system, which may be absolute (full path) or relative (starting from the current working directory).
finally block: Part of a `try` statement that executes regardless of whether an exception occurs.
Global scope: The part of a programme where variables are accessible from anywhere unless shadowed by local variables.
Keyword-only argument: A function parameter that must be specified using its name when calling the function (e.g., `func(x=10)`).
Lambda function: A small anonymous function defined using the `lambda` keyword, typically used for short, inline operations.
Local scope: Variables defined inside a function; they exist only while that function is running.
os module: A standard-library module providing tools for interacting with the operating system (e.g., directories, paths).
Parameter: A variable name inside a function definition that receives a value (argument) when the function is called.
PEP 8: The official Python style guide describing conventions for writing clean, readable, consistent code.
Relative path: A file path defined relative to the current working directory.
Return value: The output sent back by a function using the `return` statement.
Scope: The rules that determine where variables exist and how they can be accessed in a programme.
Syntax error: An error raised when Python encounters code that does not follow correct syntax (e.g., missing parentheses, incorrect indentation).
try/except: A structure used to catch and handle exceptions, preventing programmes from crashing.
Tuple unpacking: Assigning multiple values from a tuple to multiple variables in one statement (e.g., `a, b = func()`).
Whitespace (stylistic): Spaces, tabs and blank lines used to improve readability and follow style conventions such as PEP 8.
with statement: A context manager that automatically handles file opening and closing (`with open(...) as f:`).
